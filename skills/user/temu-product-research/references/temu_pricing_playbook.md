# TEMU 核价方法论 (Pricing Playbook)

> 适用于全托管模式。半托管模式定价权在卖家手里，本文档主要服务于"如何让买手选你 + 留出二次降价空间"的场景。

## 核价的本质

TEMU 买手手里有一张"同款比价表"：
- 系统自动把图片相似度 / 标题相似度 / 类目命中度高的 SKU 聚合成"同款"
- 买手看到的核价基准 = 当前同款最低供货价
- 买手的 KPI = 让买家在 TEMU 上看到全网最低价

所以新供应商报价的逻辑是：**击穿同款最低价 + 留 1–2 轮降价空间 + 仍能保毛利**。

## 一、定位"同款最低价"

1. 在 TEMU App / 网页搜索目标品的核心关键词，按"已拼数量"或"销量"排序
2. 抓 Top 5–10 的售价 `P_temu`，记录是否带运费、是否参加活动
3. 在 1688 / 拼多多 / AliExpress 反查同款源头价 `P_source`
4. 估算当前供货价：

```
P_supply_current ≈ P_temu × (1 − 平台抽成率 − 履约分摊率) − 包装/标签成本
```

> 经验值（**用前需 web_search 校准**）：
> - 平台抽成率（佣金 + 服务费）：约 0–10%
> - 履约分摊（跨境头程 + 平台尾程摊销）：低客单 25–45%、高客单 15–25%
> - 包装/标签成本：¥0.5–¥3/件

## 二、判断你是否有击穿空间

设：
- `C_factory` = 工厂出厂价（含税开票）
- `C_pack` = 包装、贴标、装箱成本
- `C_inland` = 国内到 TEMU 集运仓的运费分摊
- `C_total = C_factory + C_pack + C_inland`

报价档位：

| 报价相对 P_supply_current | 命中概率 | 风险 |
|---|---|---|
| ≥ 1.00× | 极低 | 几乎不会被选 |
| 0.90–0.99× | 中 | 可能被选作备胎，无流量倾斜 |
| 0.80–0.89× | 高 | 主选概率大，但需快速准备应对二次核价 |
| < 0.80× | 极高 | 必胜，但要确认 `C_total` 真的 cover 得住 |

> 推荐策略：**首报价 = P_supply_current × 0.85**，并预先想清楚还能再降到 `C_total × 1.10` 的下限。

## 三、核价博弈的常见套路

1. **买手要求"再降 5%"**：常态，提前在报价里预留这部分
2. **买手用更低同款施压**：要求出示对方链接，确认是否同材质同尺寸；不要被低质同款带节奏
3. **首单核价通过、二次核价砍价**：上架 2–4 周后系统会再扫一轮，要提前算清楚二次降价后的毛利
4. **跟价机制**：竞品降价你需在 24–48h 内跟降，否则流量被切走

## 四、报价示例

### 案例：硅胶折叠水杯，目标美国站

- TEMU 同款售价区间：`$2.99 – $4.99`，主流 `$3.49`
- 估算当前供货价 `P_supply_current`：

```
$3.49 × (1 − 5% 佣金 − 35% 履约) − $0.20 包装 ≈ $1.89
```

- 工厂底价 ¥7.5（≈ $1.05），包装 + 内陆 ¥1.5（≈ $0.21），`C_total ≈ $1.26`
- 击穿空间：`($1.89 − $1.26) / $1.89 ≈ 33%` → 击穿空间充足
- 推荐首报价：`$1.89 × 0.85 ≈ $1.60`
- 二次核价下限：`$1.26 × 1.10 ≈ $1.39`，对应毛利率 ≈ 9%

## 五、什么情况"宁可不做"

- 击穿空间 < 10%：上架即跟价战，红利期 < 30 天
- 二次核价下限 < `C_total`：迟早做亏
- 同款 SKU 已经超过 30 个供应商：典型红海，平台不再倾斜流量
- 该品类站内 Top 10 长期被一个工厂垄断：除非你有更深的供应链优势，否则别硬刚

## 六、与 web_search 联动的核价工作流

1. `web_search` "TEMU 平台佣金 [类目名] 2025/2026" → 拿当年的佣金表
2. `web_search` "TEMU 半托管 全托管 区别 最新" → 校准模式细节
3. `web_search` "[产品关键词] TEMU 价格" → 抓多个第三方爬虫站点的现价
4. `web_search` "[产品关键词] 1688 价格" → 估算源头底价
5. 结合 `scripts/temu_profit_calc.py` 输出最终报价档位
