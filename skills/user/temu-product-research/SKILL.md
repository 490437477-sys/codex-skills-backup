# TEMU 选品研究

围绕 TEMU 平台特性的端到端选品方法论：从机会发现 → 竞品/价格情报 → 核价与利润测算 → 物流与合规 → 风险评估 → 上架决策。本 skill 假设使用者是国内供应链卖家或跨境电商运营，重点解决 "这个品在 TEMU 上能不能做、能赚多少、怎么做" 这三个问题。

> **重要前提**：TEMU 的规则、佣金、核价标准、站点列表会持续变化。**所有费率、佣金率、扣款规则在动笔前必须用 web_search 验证最新值**，不要直接相信 skill 内的示例数字。

## 何时使用本 Skill

- 用户提供一个具体产品 / SPU / 关键词，问能不能在 TEMU 卖、利润怎么样
- 用户要在某个类目里找 TEMU 蓝海机会
- 用户对比 TEMU 与 Amazon / SHEIN / TikTok Shop 的差异和适配品类
- 用户已经在 TEMU 上架，想做竞品分析、核价复盘、改款方向

## TEMU 平台关键认知（动手前必读）

### 1. 两种主要模式

| 维度 | 全托管 (Fully Managed) | 半托管 (Semi-Managed / Y2) |
|---|---|---|
| 定价权 | 平台核价，买手主导 | 卖家自主定价 |
| 物流 | 卖家送至国内集运仓，平台负责跨境+尾程 | 卖家自备海外仓，平台触达买家 |
| 适合卖家 | 工厂型、无海外仓、SKU 多、单价低 | 有海外仓 / 美国本土发货能力、客单价中高 |
| 适合品类 | 配饰、家居小百货、季节性、低客单 | 大件、家具、家电、3C、季节性补货敏感 |
| 利润逻辑 | 拼极致供应链成本，赚规模差价 | 接近独立站逻辑，赚定价差 + 流量红利 |

> **选品的第一性问题**：先确认用户走哪条路线，因为同一个产品在两种模式下的"能不能做"答案完全不同。

### 2. 核价机制（全托管核心）

- 买手会拿同款最低价做基准，要求供货价低于基准 X%（典型 5–30%，视类目）
- 同款 / 相似款在系统中会被聚合，新供应商通常需要 **击穿现有最低价** 才能拿到流量
- 持续被压价、跟价、二次核价是常态；**报价时必须留足"二次降价空间"**
- 一旦上架卖爆，平台会引入更多供应商分流，单个 SKU 的红利期通常较短

### 3. 佣金、扣点与扣款风险

平台对卖家的成本由几部分构成（**具体百分比每年都在变，使用前用 web_search 拉最新**）：

- **平台佣金 / 服务费**：按类目，常见 0–10% 区间
- **质量扣款 / 售后赔付**：差评、破损、错发漏发的赔付往往比佣金更伤利润
- **滞销/退仓费**：全托管滞销品需自付退仓
- **自然退货成本**：欧美退货率显著高于国内，服装/鞋包尤甚（20–40% 都常见）
- **保证金/账期**：回款周期通常 T+15 至 T+30，需纳入现金流模型

### 4. 主战场与站点差异

- **美国站**：流量最大，竞争最激烈，价格最低，对包装/合规要求最严
- **欧洲（DE/FR/UK/IT/ES）**：客单价更高，对认证（CE/UKCA/RoHS/EPR）要求严
- **日韩**：客单价高、对品质和包装挑剔，适合精致小件
- **墨西哥/巴西**：客单价低、退货率高，但竞争较弱
- **中东**：增长快，注意斋月节奏与文化禁忌

> 选品时务必明确目标站点；同一个品在不同站点的天花板和合规成本可能差 10 倍。

## 工作流：6 步评估一个 TEMU 选品机会

### Step 1 — 明确输入
向用户确认（缺什么补什么）：
1. 目标产品 / 关键词
2. 模式：全托管 / 半托管 / 都看
3. 目标站点（默认美国）
4. 是否有工厂源 / 现成库存 / 海外仓
5. 预期采购成本区间

### Step 2 — 平台供需情报
用 `web_search` + `web_fetch` 收集：
- TEMU 站内同款关键词的卖家数量、价格带、销量标识（"已拼 X 件"）
- 该品类在 TEMU 是否被平台主推（首页/类目页/活动位）
- 1688/拼多多/AliExpress 上的供应链密度与底价
- Google Trends、TikTok 标签播放量验证站外热度
- Amazon 同款 BSR 与价格作为天花板参考

**输出**：需求评分 1–10、供给端拥挤度评分 1–10。

### Step 3 — 核价可行性
1. 抓 TEMU 上同款 / 近似款 Top 5 的售价 `P_temu`
2. 估算其供货价 `P_supply ≈ P_temu × (1 − 平台抽成 − 运费分摊 − 利润空间)`
3. 对比用户的工厂底价：
   - 若工厂底价 ≤ P_supply × 0.85 → 有击穿空间，可做
   - 若工厂底价 ≈ P_supply → 仅能跟价，红利期短
   - 若工厂底价 > P_supply → **不要做**，等供应链成本下来再说

> 详细公式与示例见 `references/temu_pricing_playbook.md`。

### Step 4 — 利润测算
使用 `scripts/temu_profit_calc.py` 输入成本/价格/退货率等参数，得到：
- 单件毛利
- 毛利率
- 盈亏平衡所需日均单量
- 不同退货率/扣款率下的敏感性

### Step 5 — 物流与合规
- **重量/体积**：全托管送国内集运仓，超大超重会被拒收或加费；半托管要算头程到海外仓的成本
- **认证**：电子产品（FCC/CE/PSE）、玩具（CPC/EN71）、化妆品/食品接触（FDA/LFGB）、电池（UN38.3 + MSDS）
- **品牌/IP**：卡通形象、明星肖像、注册商标一律高风险，TEMU 对侵权 0 容忍，扣款 + 下架 + 罚款
- **品类禁限售**：液体、磁铁、尖锐物、含锂电池大件、医疗器械等需逐项核实

### Step 6 — 出决策报告
按 `references/temu_decision_template.md` 模板输出：
- 综合评分 1–10
- 进 / 不进 / 谨慎进
- 推荐定价、起订量、首单数量
- 风险清单 + 对冲动作
- 后续 30/60/90 天路线图

## 评分卡（综合 1–10）

| 维度 | 权重 | 说明 |
|---|---|---|
| 站内供需差 | 25% | TEMU 上有需求且未被打烂 |
| 核价空间 | 25% | 工厂底价能否击穿现价 |
| 毛利率 | 20% | 扣完所有成本后的净利率 |
| 物流/合规可行性 | 15% | 重量、认证、IP 风险 |
| 退货/差评风险 | 10% | 品类天然退货率 |
| 复购/连带 | 5% | 是否能带动店铺其他 SKU |

加权得分：
- 8.0+：直接做，建议加大首单
- 6.5–7.9：可做，控制首单 + 准备 B 计划
- 5.0–6.4：观望，找差异化切入点再考虑
- < 5.0：不建议做
