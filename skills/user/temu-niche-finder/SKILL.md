# TEMU Niche Finder 馃徑

围绕 TEMU 平台特性，从品类热度、竞争密度、核价承受度、退货率、履约模式适配五个维度识别可做且能赚的细分机会。

## Capabilities

- **蓝海品类打分**：综合 TEMU 站内需求热度、同款卖家数量、Top 5 价格带，给出 0-100 的机会指数
- **履约模式适配判断**：根据体积重量、毛利空间、SKU 复杂度推荐全托管 / 半托管 / 本地对本地
- **核价生存力评估**：模拟 TEMU 核价机制，测算底价红线与利润缓冲
- **退货率预估**：基于类目历史退货率与品类特征，给出保守毛利率扣减建议
- **跨站点对比**：US / DE / FR / UK / JP / KR / MX / BR 同一品类的天花板差异

## When to Use

- 用户提问：在 TEMU 上卖什么、做什么品类能赚钱、TEMU 蓝海、TEMU 选品方向
- 用户给了 SPU / 产品概念 / 关键词，要求评估能不能做
- 用户已经在 TEMU 上架但利润不达标，想换品类或扩站点
- 用户对比 TEMU 与 Amazon / SHEIN / TikTok Shop 的品类适配性

## TEMU 选品关键前置知识

### 核价机制（每个品类必算）

TEMU 全托管的核心生存逻辑是核价机制：

- 同款 Top 5 供货价取最低为基准 X，TEMU 通常要求供货价 ≤ X 乘以 0.85 至 0.95
- 持续被比价、被跟价、二次核价是常态，报价时必须留 ≥ 15% 降价缓冲
- 一旦爆单，TEMU 会引入新供应商分流，单 SKU 的红利期通常 ≤ 60 天

### 三种履约模式与品类匹配

| 模式 | 适合品类 | 不适合品类 |
|---|---|---|
| 全托管 | 配饰、小家居、节庆、玩具、低客单 | 大件家具、大家电、高客单带售后 |
| 半托管 | 有海外仓、能承担售后的中大件 | SKU 极多且无海外库存 |
| 本地对本地 | 在目标国本地有现货、客单价高 | 国内供应商为主的工厂店 |

### 退货率扣减参考

TEMU 鼓励消费者退货，常见品类退货区间如下。毛利率测算时务必扣除退货损耗、退货物流成本、逆向货损。

- 服饰鞋包：20 至 35%
- 家居小件：8 至 15%
- 3C 数码：10 至 20%
- 玩具母婴：5 至 12%

## Workflow

### Step 1 — 输入确认

向用户确认五件事：目标品类或 SPU 概念、履约模式（默认全托管）、目标站点（默认 US）、是否已有供应商或工厂、期望售价区间。

### Step 2 — 平台侧情报扫描

调用 `web_search` 与 `web_fetch` 抓取：

- TEMU 同款 / 类似款的卖家数量、价格带、已拼件数
- 是否被 TEMU 主推（首页 / 品类页 / 活动坑位）
- 1688 / 多多 / AliExpress 同款供应密度与底价区间
- Google Trends、TikTok hashtag 验证站外热度
- Amazon BSR 作为天花板参考（尤其 US 站）

### Step 3 — 核价生存力测算

抓 TEMU Top 5 同款供货价，去掉最低求均价 P_avg，预设核价系数 0.9：

```
P_supply_max = P_avg 乘以 0.9 乘以 (1 - 平台扣点 - 履约成本占比)
```

如果用户成本 > P_supply_max，输出「核价红线告警」，建议调整产品或供应链。

### Step 4 — 退货率与利润模型

按品类历史退货率扣减毛利，加入退货物流、逆向货损、平台质量赔付预估，给出净利润率。

### Step 5 — 跨站点天花板上探

针对 US / DE / FR / UK / JP / KR / MX / BR 等市场，分别估算客单价上限、合规门槛（CE / UKCA / FCC / PSE / KR 认证）、文化与季节性差异。

### Step 6 — 机会评分与决策

汇总五维评分（需求 / 竞争 / 核价 / 退货 / 站点），输出 0-100 总分：

- ≥ 75：建议立即进入，分配预算
- 60-75：可试款，控制库存风险
- 45-60：观察期，不建议首发投入
- < 45：放弃或调整品类

## Quick Mode

只回答「在 TEMU 上能不能做 X」时，输出 4 行：

1. 模式推荐（全托管 / 半托管 / 不建议）
2. 核价红线供货价（USD）
3. 预估退货率与净利润率
4. 机会评分加一句话决策

注意：核价系数、平台扣点、退货率随类目与时间变化，输出前必须用 `web_search` 拉最新平台公告校准。
