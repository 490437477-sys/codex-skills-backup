# TEMU 核价机制详解 (Pricing Mechanics)

> 适用: 全托管 / 半托管卖家。TEMU 的核价逻辑直接决定店铺生死，本文件拆解平台自动比价、强制降价、底价保护、退货扣款、质保金、节庆加价空间与佣金结构。

## 一、核价机制的底层逻辑

TEMU 母公司 Pinduoduo 在国内已经把"全网最低价"做成基础设施，跨境延续同一套算法:

| 层级 | 触发条件 | 后果 |
|---|---|---|
| L1 同款比价 | 同图、同标题、SPU 一致 | 价格超出中位数即标记"价格力弱" |
| L2 类目对标 | 同叶子类目 Top 100 | 价格高于 75 分位即限流 |
| L3 平台竞对 | 同步 Amazon / Walmart / SHEIN | 同款差价 > 15% 触发核价工单 |

平台 KPI 是"让买家在 TEMU 上看到全网最低价"，所以新链接需要穿同款最低价 + 留 5%-10% 降价空间 + 仍能保本。

## 二、自动比价系统的运作

### 2.1 触发频率
- 新品上架 7 天内: 每 12 小时跑一次
- 稳定期: 每 24-48 小时一次
- 大促前 30 天: 每 6 小时一次
- 每次核价后商家中心推送"价格预警"工单

### 2.2 比价锚点
- `P_market_low`: 同款全网最低价 (Amazon / Walmart / 1688 反查)
- `P_category_p25`: 同叶子类目 25 分位
- `P_internal_low`: TEMU 站内同款最低 SKU

### 2.3 评分公式

```
price_score = 100 × (P_market_low / P_your) ^ 0.6
            + 30 × (P_category_p25 / P_your) ^ 0.3
            − 50 × max(0, P_your − P_internal_low) / P_internal_low
```

> price_score ≥ 80 给搜索权重 + 流量倾斜；< 50 直接隐藏搜索结果。

## 三、强制降价机制

### 3.1 三级触发
| 级别 | 工单类型 | 时限 | 不响应后果 |
|---|---|---|---|
| P1 价格力预警 | 软性建议降价 | 48h | 搜索降权 |
| P2 强制核价 | 强制改价 | 24h | 链接下架 |
| P3 同款违规 | 判定为高仿或未授权 | 12h | 永久下架 + 扣质保金 |

### 3.2 降价分摊规则
- 首次核价: 商家承担 100% 降价
- 第二次核价: 商家 70% + 平台补贴 30%
- 大促期间核价: 平台补贴可达 50%
- 跟价时间窗: 订单生成到发货前 24 小时内若同款降价，平台会主动给买家发差价退款券，由商家承担

## 四、底价保护 (Floor Price)

### 4.1 平台允许的最低售价
```
P_floor = max(C_landed × 1.08, C_total × 1.15, P_market_low × 0.92)
```

> 含义: 平台不会要求跌破 C_total + 15% 毛利，但也不会允许高于市价 8%。

### 4.2 启用与锁定
- 启用条件: 店铺评分 ≥ 4.8 (近 30 天) + 类目退货率 < 行业均值 + 链接近 7 天日均订单 > 10
- 锁定方式: seller.meridian.tera.com 后台勾选"价格保护"
- 锁定后 14 天内不再触发 L1 比价，每 30 天可申请一次延长

## 五、退货扣款 (Return Deduction)

### 5.1 退货类型与扣款

| 退货原因 | 扣款方 | 扣款比例 | 商家承担 |
|---|---|---|---|
| 买家无理由 | TEMU | 0 | 仅承担库存损失 |
| 描述不符 | 商家 | 100% | 退款 + 运费 + 质检费 |
| 质量问题 | 商家 | 100% | 退款 + 退运 + 销毁费 |
| 物流破损 | 平台 / 物流商 | 50/50 | 视合同 |
| 错发漏发 | 商家 | 100% | 退款 + 重发成本 |

### 5.2 健康分公式与影响
```
health_score = 100 − 0.3 × return_rate × 100 − 0.5 × complaint_rate × 100
```

> return_rate > 8% 进入"黄牌"名单，触发额外抽检。全托管由 TEMU 海外仓质检二次销售，半托管原路退回，商家承担跨境运费 + 销毁费。

## 六、质保金 (Quality Deposit)

### 6.1 缴纳规则
- 新店入驻: 人民币 10,000 元 (或等值美元)
- 类目额外保证金: 美妆 / 食品 / 母婴 + 50,000 元
- 注销店铺: 90 天后退还 (无未结售后)

### 6.2 扣罚场景
| 违规 | 扣罚金额 |
|---|---|
| 假冒伪劣 | 全额扣除 + 3 倍罚款 |
| 严重质量事故 | 5,000 - 30,000 元 |
| 知识产权侵权 | 10,000 - 50,000 元 |
| 虚假发货 | 2,000 元 / 次 |

## 七、节庆加价空间

| 节庆 | 提价上限 | 持续时间 |
|---|---|---|
| Black Friday | 5% | 11/20 - 12/2 |
| Christmas | 8% | 12/15 - 12/24 |
| 母亲节 | 10% | 5/1 - 5/12 (美国) |
| 父亲节 | 8% | 6/15 - 6/21 |
| Back-to-School | 5% | 7/15 - 9/5 |
| Easter | 5% | 3/15 - 4/9 |
| 当地国庆 (各市场) | 3%-5% | 1-3 天 |

- 提价申报: 在 seller portal 营销活动 → 节日提价申请 提交；需提供竞品价格截图 + 提价后毛利率测算；审核周期 24-48 小时
- 不允许提价的节庆: 双 11 / 双 12 / 618 必须降价 ≥ 15%；平台周年庆必须降价 ≥ 20%

## 八、平台佣金与扣点

### 8.1 全托管
```
平台扣点 = GMV × (5% − 10%) + 类目基础服务费
```

- 类目基础服务费 0.5% - 3% + 支付手续费 1% + 提现手续费 1% (单笔最低 $1)
- 实际净到手 ≈ 售价 × (1 − 14%) − 物流成本

### 8.2 半托管
```
平台扣点 = GMV × (8% − 15%) + 广告费另算
```

- 物流扣点 5% - 8% (按重量阶梯) + 类目佣金 3% - 7%
- 实际净到手 ≈ 售价 × (1 − 18%) − 自有海外仓成本

### 8.3 本地对本地 (Local-to-Local)
- 平台抽佣 8% - 12%，商家自负责物流 / 售后 / 退货
- 净到手最高，但运营复杂度也最高

## 九、报价实战公式

### 9.1 反推底价
```
C_total = C_factory + C_pack + C_inland + C_customs + C_platform_commission
P_break_even = C_total × 1.10        // 留 10% 最低毛利
P_safe = max(P_break_even, P_floor)
P_first_quote = P_safe × 0.85        // 留 15% 降价空间
```

### 9.2 红线
- `P_first_quote / C_total < 1.20` → 别上: 二次核价必亏
- `C_platform_commission > 15%` → 考虑转半托管或 Local-to-Local
- 类目大盘退货率 > 12% → 质保金扣罚风险高，建议绕开
- 节庆定价速算: BF `P_normal × 0.95`、母亲节 `P_normal × 1.10`

## 十、避坑清单

- 不要在标题 / 主图隐藏 SKU: 算法识别后会触发 L3 同款违规
- 不要用低价冲量后涨价: 平台保留历史价，涨价幅度 > 30% 直接限流
- 不要拒接 P1 工单: 累计 3 次未响应会被标记消极卖家
- 不要在多个 SKU 重复同一主图: 算法判定重复铺货会批量下架
- 节庆前 30 天不要大改主图: 改图会被系统判定上新，重新进入 7 天观察期
