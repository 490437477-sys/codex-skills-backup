# Ozon PPC 出价策略与 ДРР 公式

## ДРР 定义

**ДРР** = Доля рекламных расходов(广告支出占广告收入比)

```
ДРР = (Рекламный расход / Выручка от рекламы) × 100%
```

**本质上等同 Amazon ACoS。** 区别:
- ACoS 是 Amazon 英文术语
- ДРР 是 Ozon / 俄罗斯市场通用俄语术语
- 计算公式和含义完全相同

**与 TACoS 的区别**:
- **ДРР** = 广告花费 / 广告带来的收入
- **TACoS** = 广告花费 / 店铺**总**收入(含自然单)
- TACoS 是更宏观的健康指标

## 关键公式

### 1. Break-even ДРР(盈亏平衡 ДРР)

最大可承受的 ДРР,超过即亏本:

```
Break-even ДРР = (毛利 / 售价) × 100%

示例:
售价 ₽1500,商品成本 ₽625,Ozon佣金 ₽180,FBO物流 ₽95,退货预留 ₽75
毛利 = ₽1500 - ₽625 - ₽180 - ₽95 - ₽75 = ₽525
Break-even ДРР = 525 / 1500 × 100% = 35%
```

> **注意**: НДС(VAT)处理 — Ozon 作为 tax agent 代扣 20% НДС,卖家实际收到的是**不含 VAT 的金额**。Break-even 计算应基于卖家实际收到的金额。

### 2. Max CPC 上限

在目标 ДРР 和转化率下的最高可承受每次点击成本:

```
Max CPC = (目标 ДРР × 售价 × 转化率) / 100

示例:
目标 ДРР 25%,售价 ₽1500,转化率 2%
Max CPC = (25 × 1500 × 0.02) / 100 = ₽7.5/click
```

> **重要**: 这是 **Max**,不是建议。实际出价应低于此值。

### 3. 利润公式

```
单件净利润 = (售价 × (1 - VAT%)) - 商品成本 - 佣金 - 物流 - 退货预留 - 广告分摊

其中广告分摊 = (售价 × ДРР%) / 订单量
```

### 4. 预算公式

```
日预算 = (目标订单数 / 30 × CPC) / 转化率

示例:
目标月 60 单,平均 CPC ₽30,转化率 2%
日预算 = (60/30 × 30) / 0.02 = ₽3,000/天
```

## 生命周期 ДРР 目标

| 阶段 | 目标 ДРР | 容忍 ДРР | 重点 |
|---|---|---|---|
| **Launch**(0-30 天) | 30-50% | <70% | 抢曝光、收集数据 |
| **Growth**(30-90 天) | 15-25% | <35% | 规模化盈利词 |
| **Mature**(90+ 天) | 8-15% | <20% | 防守 + 稳定盈利 |
| **Defense**(竞品冲击) | 5-10% | <15% | 拦截竞品 |

## 出价策略类型

Ozon Performance 支持 **3 种自动出价策略**:

### 1. Максимум показов(Max Impressions)

- Ozon 自动调整出价以获得最多展示
- 适合:**Brand awareness / 新品冷启动**
- 风险:可能烧光预算,转化率未知时谨慎

### 2. Минимальная цена(Lowest Price)

- Ozon 自动降到刚好能展示的位置
- 适合:**Мature 阶段的防守**
- 风险:展示量小,数据样本不足

### 3. 手动 Максимальная цена клика(Manual Max CPC)

- 卖家完全控制
- 适合:**Launch + Growth 阶段** — 推荐用于精细化运营
- 建议:每个 ad group 单独出价,避免全局一刀切

## Bid 调整规则

### 通用规则(基于 ДРР 表现)

| 当前 ДРР vs 目标 | 调整 | 备注 |
|---|---|---|
| ДРР < 50% 目标 | +20-30% | 表现优秀,放大 |
| ДРР 50-80% 目标 | +10-15% | 还行,小步快推 |
| ДРР 80-100% 目标 | 保持 | 接近目标,稳定 |
| ДРР 100-130% 目标 | -10-20% | 略亏,小降 |
| ДРР > 130% 目标 | -30-50% 或暂停 | 严重亏本 |

### 关键词级别规则(基于点击与转化)

| 状态 | 动作 |
|---|---|
| 30+ 点击 + 0 订单 | **NEGATE** |
| 20+ 点击 + 1 订单 | 保持 / 微降 |
| 20+ 点击 + 2+ 订单 | **迁移到 Поиск Exact** + 提高出价 |
| 10+ 点击 + 5+ 订单 | **扩展匹配词 + 同义词** |
| ДРР > 2× 目标 | 降出价 30% |

## НДС(VAT)处理

**关键认知**: Ozon 作为 **налоговый агент(tax agent)** 在俄罗斯 B2C 销售中代扣代缴 НДС(增值税)。

### 影响范围

| 项目 | 影响 |
|---|---|
| 标价 (цена) | 含 VAT(20% 已计入) |
| Ozon 佣金 | 按**含税**价计算 |
| 广告费 | 按**含税**金额扣费 |
| 卖家实际收到 | **售价 × (1 - 20%)** = 不含 VAT 净额 |
| 广告费对账单 | 显示含 VAT 总额 |

### 实际案例

假设商品 ₽1000,Ozon 佣金 12%,广告花 ₽100:

```
买家支付: ₽1000 (含 VAT)
Ozon 代扣 VAT: ₽200 (转交税局)
卖家净额(不含 VAT): ₽800

Ozon 佣金: ₽1000 × 12% = ₽120
广告费: ₽100
总扣除: ₽220
卖家实收: ₽800 - ₽220 = ₽580

商品成本: ₽300(假设)
净利润(不含广告前): ₽580 - ₽300 = ₽280
ДРР: ₽100 / ₽1000 = 10%
净利率: ₽280 / ₽1000 = 28%
```

> **利润建模脚本**: 用 `ozon-product-research` 中的 `ozon_profit_calc.py` 做精确建模。

## 验证清单

每次调整出价前:
- [ ] 已用 `web_search` 验证当前类目 CPC 中位数
