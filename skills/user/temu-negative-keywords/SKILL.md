# TEMU Negative Keywords 馃徑

通过否定关键词与否定商品管理，把 TEMU Ads 预算从无关流量上撤出，保护高转化核心词，实现 ACOS 下降与单量稳定。

## Capabilities

- **搜索词报告挖掘**：从 TEMU Ads 后台拉近 30 / 60 / 90 天搜索词数据
- **低效词识别**：按曝光 / 点击 / 转化 / ACOS 四象限分类
- **否定匹配策略**：精准否定（exact）与词组否定（phrase）按场景搭配
- **跨站点词库共享**：把 US 站验证过的否定词复用至 DE / UK / FR 等多语种
- **节庆临时屏蔽**：黑五 / 圣诞 / 母亲节后清理活动期爆量但无转化的词
- **防误伤机制**：核心高转化词加白名单、监控否定后曝光跌幅

## When to Use

- 用户跑 TEMU Ads 一段时间后 ACOS 居高不下，想清理投放
- 用户准备大促，需要预先把明显无关词加入否定
- 用户做新站点冷启动，需要从零搭建否定词库
- 用户发现某 SKU 退货率异常高，需要排查是不是投错人群

## TEMU Ads 否定逻辑前置

### 匹配机制

| 匹配方式 | 触发规则 | 适用场景 |
|---|---|---|
| 精准否定 | 用户搜索词完全等于否定词 | 完全无关的错位词（如 iphone case 投到 android cable） |
| 词组否定 | 用户搜索词包含否定词 | 无关品类词（如 charger 包含 battery charger） |
| 广泛否定 | 默认行为，不专门设置 | 不建议主动加广泛否定 |

> 注意：TEMU Ads 的匹配机制持续微调，上线前用 `web_search` 校准当前规则。

### 四象限判定

按 30 天数据归类搜索词：

- 高曝光高转化：保留并提价
- 高曝光低转化：加词组否定
- 低曝光高转化：精准否定其无关变体
- 低曝光低转化：精准否定

## Workflow

### Step 1 — 数据采集

从 TEMU Ads 后台导出：

- 近 30 / 60 / 90 天搜索词报告（曝光、点击、转化、ACOS）
- 分广告组 / 分关键词维度的花费分布
- 关联 SKU 的退货率与差评率

### Step 2 — 低效词识别

按四象限规则标注每条搜索词：

- 曝光 ≥ 1000 且 CTR ≥ 2% 但转化 = 0：高曝光低转化候选
- 点击 ≥ 50 且 CPC 偏高但 ACOS > 100%：点击浪费候选
- 关联 SKU 退货率 > 类目中位数 1.5 倍：人群错配候选

### Step 3 — 否定策略决策

为每个候选词选择否定方式：

- 完全错位（如品牌词窜入）：精准否定
- 部分相关但转化差（如泛词 big size 投到具体款式）：词组否定
- 临时活动词（如 prime day）：节庆后清理，不入长期词库

### Step 4 — 跨站点词库搭建

US 站积累的英文否定词可直接复用至 UK / CA / AU 英文市场。

DE / FR / ES / IT 等语种需本地化翻译：

- 利用 `web_search` 查该语种的同义错位词
- 注意单复数、阴阳性、动词变位
- 每个新站点首月每周回看一次搜索词

### Step 5 — 防误伤

加否定前必须做的三件事：

- 查该词历史转化订单数（≥ 5 单才考虑保留）
- 查该词关联 SKU 的销售额占比
- 监控加入否定后 48 小时曝光跌幅（跌幅 > 20% 立刻撤掉）

### Step 6 — 词库维护节奏

| 周期 | 动作 |
|---|---|
| 每周 | 清理明显错位词 |
| 每月 | 完整跑一遍搜索词报告 |
| 每节庆后 | 清理活动期临时词 |
| 每季度 | 复核长期词库，撤掉已不再适用的旧否定 |

## Output

| 字段 | 内容 |
|---|---|
| 否定词 | 关键词原词 |
| 匹配方式 | 精准 / 词组 |
| 加入日期 | YYYY-MM-DD |
| 所属广告组 | 广告组名称 |
| 上线前 ACOS | 该词触发时 ACOS |
| 预计节省花费 | 月度估算 |
| 关联 SKU | 商品链接 |

## Quick Mode

只回答「这个搜索词要不要否定」时，输出 3 行：

1. 判断（保留 / 词组否定 / 精准否定）
2. 一句话理由
3. 若否定，预计月节省花费区间

注意：TEMU Ads 否定规则与后台字段名会迭代，决策前用 `web_search` 拉最新卖家中心说明。
