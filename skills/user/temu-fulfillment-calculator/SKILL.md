# TEMU Fulfillment Calculator

全托管 / 半托管 / 本地对本地三模式对比 + 决策树推荐，针对卖家「有没有海外仓」「客单价高低」「SKU 数量」「目标市场」给出最优履约模式与混合策略。

> **重要前提**：TEMU 履约模式细节（佣金比例、回款时效、合规要求）随政策更新。**所有费率与门槛在动笔前必须用 web_search 验证最新值**，不要直接套用历史示例数字。

## Capabilities

- 三种履约模式 8 维度并列对比（库存归属、发货地、定价权、退货承担、结算账期、佣金扣点、回款速度、合规风险）
- 基于 6 个卖家条件输入（SKU 数、客单价、有无海外仓、目标市场、合规资质、退货率）输出最优模式
- 混合模式策略：核心 SKU 走 A 模式，长尾 SKU 走 B 模式
- 模式切换成本估算：从全托管 → 半托管需要补齐哪些能力（海外仓 / 尾程 / VAT）
- 各市场专属建议：US / EU / JP / KR / MX / BR 哪个模式最优

## When to Use

- 用户问「TEMU 全托管和半托管怎么选」「哪个更适合我」
- 用户问「TEMU 半托管是什么」「TEMU Y2 是什么意思」
- 用户问「TEMU 本地对本地怎么做」「要不要建海外仓」
- 已有工厂型卖家想升级到半托管或本地对本地
- 已有海外仓的贸易商想搞清楚怎么上 TEMU
- 用户对比 TEMU vs Amazon FBA / TikTok Shop 履约

## TEMU 三种履约模式速览

| 维度 | 全托管 Fully Managed | 半托管 Semi-Managed (Y2) | 本地对本地 Local-to-Local |
|---|---|---|---|
| 库存归属 | 平台仓 | 卖家海外仓 | 卖家本地仓 |
| 发货地 | 国内 → TEMU 集运仓 | 海外仓直发 | 本地仓同城发 |
| 定价权 | 平台主导核价 | 卖家自主定价 | 卖家自主定价 |
| 平台佣金 | 5% - 10% | 8% - 15% | 5% - 12% |
| 退货承担 | 商家（货损+回程） | 商家 | 商家 |
| 结算账期 | T+15 ~ T+30 | T+7 ~ T+15 | T+3 ~ T+7 |
| 履约时效 | 7 - 15 天 | 3 - 7 天 | 1 - 3 天 |
| 适合卖家 | 工厂型 / 无海外仓 | 贸易商 / 有海外仓 | 当地卖家 / 大卖 |
| 适合品类 | 低客单 / 轻小件 / SKU 多 | 中高客单 / 大件 / 需快时效 | 全品类 / 重合规 |

## Workflow

### Step 1 - 采集卖家画像
向用户收集 6 个条件：
1. SKU 数量（< 50 / 50 - 500 / > 500）
2. 客单价区间（< USD 5 / 5 - 20 / > 20）
3. 是否已有海外仓（无 / 美国 / 欧洲 / 都有）
4. 目标市场（US / EU / JP / KR / MX / BR 等）
5. 是否已有合规资质（VAT / EPR / FCC / PSE 等）
6. 历史退货率（< 10% / 10% - 25% / > 25%）

### Step 2 - 模式匹配决策树
按优先级判断：
- 有本地海外仓 + 客单价 > USD 20 → 优先「本地对本地」
- 有海外仓 + 客单价 USD 5 - 20 → 优先「半托管」
- 无海外仓 + SKU 多 → 优先「全托管」试水
- SKU 少 + 高客单 → 优先「半托管」（利润率更高）

### Step 3 - 单 SKU 利润对比
用同一个 SKU 分别测算三模式的净利率，给出数字差异而非抽象判断。

### Step 4 - 模式切换成本估算
若推荐「半托管」但卖家当前是全托管，列出切换需要补齐的能力：海外仓合作、尾程账号、VAT 注册、ERP 接入，估算一次性投入金额。

### Step 5 - 混合策略建议
对成熟大卖，建议核心爆款走「半托管」（利润高 + 时效快），长尾 SKU 走「全托管」（省心）。

### Step 6 - 风险点标注
每个推荐都附带 3 - 5 个核心风险点（如：核价、库存、退货、合规、税务）。

## Output

输出主表：`模式 | 库存 | 发货 | 定价权 | 佣金 | 回款 | 时效 | 适合场景`

附表：
- 推荐模式 + 推荐理由（3 - 5 条）
- 单 SKU 三模式净利率对比
- 模式切换一次性投入估算
- 风险点清单（按发生概率排序）

## Quick Mode

只输入：是否有海外仓 + 客单价 + 目标市场。3 个条件 → 直接输出推荐模式与核心利弊。


## 实战 Tips

- **永远跑一次 Quick Mode 再做详细测算**：5 个数字 10 秒出结果，先判断大方向值得做再深入。
- **TEMU 数字每月变**：所有费率 / 扣点 / 核价基线 / 退货率每月都可能微调，建议每 30 天复核一次。
- **多市场分散风险**：单一市场被核价压垮时可以切换到另一市场，履约链路要支持多目的地。
- **保留 10% - 15% 利润缓冲**：所有测算都假设最好情况，实际留出 10% - 15% 的缓冲空间才稳。
