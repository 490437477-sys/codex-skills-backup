# noon Rich Content

Build compelling bilingual Enhanced Brand Content for the noon marketplace. noon Rich Content is the closest analogue to Amazon A+ Content in MENA: brand-registered sellers can use modules to tell a richer product story.

## Capabilities

- **Brand registry check**: confirm the seller is enrolled in noon Brand Registry before recommending Rich Content modules
- **Module selection**: choose from noon Rich Content module types (hero, comparison, lifestyle, feature grid, brand story)
- **Bilingual copy**: write Arabic-first (RTL) + English copy in matching tone; Arabic is the dominant language for noon.sa
- **RTL layout guidance**: recommend image compositions that read correctly right-to-left
- **Comparison chart planning**: build feature-vs-feature matrices that resonate with MENA shoppers (e.g. halal, family-size, voltage 220V)
- **Image briefs**: detailed art direction briefs for designers (composition, color palette, typography, model demographics)
- **Cultural sensitivity**: avoid imagery that conflicts with MENA norms; emphasize family, modest fashion, premium feel
- **Mobile-first design**: 70%+ of noon traffic is mobile; modules must render well on small screens

## Workflow

### 1. Confirm brand registry
Rich Content requires brand registry on noon. If the seller is not enrolled, recommend enrollment first (3-6 weeks).

### 2. Pick the module mix
Default 5-module layout:

| # | Module | Purpose |
|---|--------|---------|
| 1 | Hero banner | Brand promise + lifestyle |
| 2 | Feature grid (4-tile) | Key product features with icons |
| 3 | Comparison chart | You vs competitors / old vs new |
| 4 | Lifestyle image | Product in use (MENA setting) |
| 5 | Brand story | Heritage, mission, customer trust |

Add more modules only if conversion data supports it.

### 3. Write bilingual copy
- Arabic (primary for noon.sa, optional for noon.ae/en-EG)
- English (secondary, used for expat / English-first shoppers)
- Keep bullet length ≤ 80 characters (mobile truncation)
- Use strong action verbs, family-friendly tone

### 4. Build image briefs
For each module:
- Dimensions (e.g. hero 1920x720)
- Subject matter
- Color palette (align with brand, consider gold/black/red for premium MENA feel)
- Model demographics if applicable
- Text overlay space (leave room for Arabic on right side for RTL balance)

### 5. Cultural review
Avoid:
- Mixed-gender physical contact imagery
- Alcohol, pork references
- Off-season Western holiday imagery (Christmas, Halloween)
- Non-halal certification claims unless verified

Prefer:
- Family settings
- Home / hospitality / gifting contexts
- Modest fashion presentation
- Gold / navy / white color palettes

### 6. Mobile QA
Test each module at 360px width. Text must remain readable; imagery must not crop awkwardly.

## Output

- **5-module layout plan** with bilingual copy per module
- **Image briefs** (one per module, designer-ready)
- **Cultural compliance checklist** (passed / flagged / action required)
- **Mobile QA notes**

## Quick Mode

If user wants only one module (e.g. comparison chart): return just that module's copy + image brief.
