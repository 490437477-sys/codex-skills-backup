# TEMU Sales Estimator

基于 BSR / 类目排名 / 列表位 / 关键词热度 / 广告投放 / 季节性 / 平台流量分配规则，估算月销量与营业额，针对 TEMU 特有的「平台强流量干预」「核价奖励流量」「高退货率」机制做修正。

> **重要前提**：TEMU 流量分配规则、类目榜单、节日促销节奏每年都在变。**所有 BSR 数据、类目容量、节日前瞻在动笔前必须用 web_search 验证最新值**，不要直接套用历史示例数字。

## Capabilities

- 基于 BSR / 类目排名估算月销量与月营业额（按品类分档）
- TEMU 自然流量 vs 广告流量占比分析（默认自然 60% - 80%，广告 20% - 40%）
- 季节性修正系数：Black Friday、Christmas、母亲节、父亲节、Easter、Back-to-School
- 三模式销量差异：全托管 vs 半托管 vs 本地对本地（履约时效对转化率的影响）
- 核价对销量的杠杆：降价 5% / 10% / 15% 通常能带来多少额外单量
- 「值不值得做」销量门槛判断：低于 X 单 / 月 不建议上

## When to Use

- 用户问「这个 SKU 在 TEMU 上能卖多少」「TEMU 月销多少」
- 用户问「TEMU 流量怎么分配」「要不要投广告」
- 用户问「TEMU 哪个节日最好卖」「黑五 / 圣诞能做多少」
- 用户准备选品需要先估算天花板销量
- 用户已经上架但销量低，想判断是品的问题还是流量的问题
- 用户问「TEMU 类目排名怎么看」「TEMU BSR 准吗」

## TEMU 流量分配规则（与 Amazon 差异巨大）

- **TEMU 是中心化流量**：平台决定谁上首页、谁进推荐位，卖家流量获取能力远弱于 Amazon
- **核价即流量**：愿意降价的卖家会被平台推到更好位置，相当于「用利润换流量」
- **广告位有限**：TEMU Ads 仍在快速发展，搜索广告 + 推荐位为主，CPC 比 Amazon 低
- **新卖家冷启动难**：没有历史评价的 SKU 自然转化率低，必须靠广告 + 活动起量

## Workflow

### Step 1 - 采集 SKU 基本信息
1. 类目（服饰 / 家居 / 3C / 母婴 / 美妆 / 户外 等）
2. 目标市场（US / EU / JP / KR / MX / BR）
3. 履约模式（全托管 / 半托管 / 本地对本地）
4. 上架价 / 目标售价
5. 是否有广告预算
6. 上架时间节点（旺季前 X 周）

### Step 2 - 类目天花板估算
按类目给出月销区间（保守 / 乐观）：
- 服饰类：月销 1k - 50k 件
- 家居小件：月销 500 - 20k 件
- 3C 配件：月销 1k - 30k 件
- 美妆个护：月销 500 - 15k 件
- 母婴用品：月销 300 - 10k 件

### Step 3 - BSR / 列表位反推
如果用户提供类目排名（Top 100 / Top 1000 / Top 10000），反推日均销量。

### Step 4 - 季节性系数叠加
按上架时间点叠加季节系数：

| 节日 / 节点 | 提前备货周数 | 销量峰值系数 |
|---|---|---|
| Black Friday + Cyber Monday | 6 - 8 周 | 3 - 5x |
| Christmas | 6 - 8 周 | 2 - 4x |
| Mothers Day | 3 - 4 周 | 1.5 - 2x |
| Fathers Day | 3 - 4 周 | 1.3 - 1.8x |
| Easter | 3 - 4 周 | 1.2 - 1.6x |
| Back-to-School | 4 - 6 周 | 1.5 - 2.5x |
| Prime Big Deal Days | 5 - 7 周 | 2 - 4x |

### Step 5 - 模式销量修正
- 全托管：履约时效 7 - 15 天，转化率基准 1
- 半托管：履约时效 3 - 7 天，转化率基准 1.2 - 1.4
- 本地对本地：履约时效 1 - 3 天，转化率基准 1.3 - 1.6

### Step 6 - 广告与核价杠杆
给出广告预算与核价两种「加流量」手段的预期销量弹性。

### Step 7 - 销量 vs 利润平衡
结合 profit-analyzer 输出「销量 × 单件利润」的最优组合。

## Output

输出主表：`类目 | 月销保守 | 月销乐观 | 月营业额 | 利润率 | 月净利`

附表：
- 季节性销量曲线（按周分布）
- 流量来源拆解（自然 / 广告 / 推荐位）
- 模式修正后的最终月销预测
- 「值不值得做」销量门槛判断（绿灯 / 黄灯 / 红灯）

## Quick Mode

只输入：类目 + 售价 + 模式 + 市场。4 个条件 → 直接输出月销区间与月营业额区间。

## 常见误区

- **把 Amazon BSR 直接套到 TEMU**：TEMU 类目划分不同，相同 BSR 名次销量差异可达 3 - 5 倍。
- **忽略核价杠杆**：TEMU 卖家主动降 5% - 10% 通常能换来 30% - 80% 的额外单量。
- **只看月销不看退货**：TEMU 月销 10k 但退货率 25%，实际到手只有 7.5k。
- **旺季前 1 周才备货**：旺季备货周期需要 6 - 8 周，临时上架来不及吃红利。



## 实战 Tips

- **永远跑一次 Quick Mode 再做详细测算**：5 个数字 10 秒出结果，先判断大方向值得做再深入。
- **TEMU 数字每月变**：所有费率 / 扣点 / 核价基线 / 退货率每月都可能微调，建议每 30 天复核一次。
- **多市场分散风险**：单一市场被核价压垮时可以切换到另一市场，履约链路要支持多目的地。
- **保留 10% - 15% 利润缓冲**：所有测算都假设最好情况，实际留出 10% - 15% 的缓冲空间才稳。
