# MCP Server 修复记录 - 2026-07-03

## 问题

`uvx 0.11.6` 不再支持 `--from <pkg>` 这种空格分隔的参数形式。
如果用 `--from mcp-tavily`,uvx 会把整个参数忽略,然后报:

```
Provide a command to run with `uvx <command>`.
```

`uvx` 根本不会去拉 `mcp-tavily` 包,直接退出了。

## 修复

把 `config.toml` 里所有 `uvx --from X ...` 改成 `uvx --from=X ...`:

### tavily 段

```diff
[mcp_servers.tavily]
command = "uvx"
- args = ["--from", "mcp-tavily", "--with", "python-dotenv", "python", "-m", "mcp_server_tavily"]
+ args = ["--from=mcp-tavily", "--with=python-dotenv", "python", "-m", "mcp_server_tavily"]
```

### minimax 段

```diff
[mcp_servers.minimax]
command = "uvx"
- args = ["--from", "minimax-coding-plan-mcp", "minimax-coding-plan-mcp", "-y"]
+ args = ["--from=minimax-coding-plan-mcp", "minimax-coding-plan-mcp", "-y"]
```

## 验证

```powershell
cmd /c "uvx --from=mcp-tavily --with=python-dotenv python -m mcp_server_tavily"
# 启动后挂 stdio,无输出 = 正确
```

## 适用范围

- `uv >= 0.5`
- 主要影响 `mcp_servers.*` 用 `uvx` + `--from` 的段
- 本机 `config.toml` 已修复(2026-07-03)
- 仓库脱敏 `config.toml.sanitized` 用的是 `npx` 路线,不受影响

## 相关

- tavily 段:装好后 fallback 有 `tavily_npm` 段(npm 版),任一可用即可
- minimax 段:唯一 MCP 入口
